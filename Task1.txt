eventLoop (событийный цикл) - непрерывный цикл выполнения кода из очереди задач
(главного потока), примерный алгоритм:
1. Ожидание
2. Если есть задачи в очереди макрозадач - выполнить старейшую 
3. Если есть задачи в очереди микрозадач - выполнить все по принципу FIFO
4. Перейти к выполнению следующей макрозадачи
5. Если очереди пустые - ожидание

Main Thread (главный поток) - единственный поток, в котором возможны манипуляции с DOM. 
Это исключает обращение к одному и тому же элементу DOM из разных потоков одновременно

Процесс загрузки и отображения страницы в браузере:
1. Загрузка и чтение HTML, построение DOM
2. Загрузка и чтение CSS и JS, построение CSSOM
3. Событие DomContentLoaded, построение дерева рендеринга,
   выполнение JS и CSS.
   * здесь JS блокирует дальнейшую загрузку и отрисовку
   * это нужно иметь в виду при оптимизации производительности
   * в частности грамотно использовать атрибуты async и defer
4. Построение макета страницы
5. Конечный вид, "окрашивание" макета страницы
6. Событие Load
